## dmg 问题
- `hdiutil makehybrid` 打包 .app bundle 后打开只显示空的 Contents 文件夹（不正确）
- `hdiutil create -format UDZO -srcfolder` 需要挂载磁盘镜像，沙箱权限不允许
- **最终方案**：`hdiutil makehybrid -hfs` 从 /tmp staging 目录创建（含 app + Applications 快捷方式），12MB，正常可用

## 前端文件同步问题（重要！）
- **根因**：Vite 从 `client/` 根目录读取 index.html/main.js/styles.css，但我们一直只编辑了 `client/src/` 下的文件
- 导致之前所有构建的 app 都包含旧版前端代码（没有昵称输入框等 v1.1.0 功能）
- **修复**：将 `client/src/` 下的 index.html、main.js、styles.css 同步到 `client/` 根目录
- 以后修改前端代码时**必须同时更新两处**，或直接在根目录编辑

## JS 打包问题（致命 bug！）
- **根因**：HTML 中 `<script src="main.js">` 不是 `type="module"`，Vite 不打包非 module 脚本
- 导致 dist 目录只有 CSS 没有 JS，app 所有按钮完全无法点击
- **修复**：改为 `<script type="module" src="main.js">`，Vite 正确打包 JS 到 dist/assets/
- 最新 dmg (4.4MB) 由 Tauri 的 bundle_dmg.sh 自动生成（这次没被沙箱拦截）
