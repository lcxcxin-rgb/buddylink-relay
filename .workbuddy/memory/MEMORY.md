# BuddyLink Project Memory

## Project Overview
BuddyLink - Mac desktop widget app for paired Mac-to-Mac interaction across networks.
- Tech: Tauri v2 (Rust + WebView), Node.js WebSocket relay server, E2EE encryption
- Features: pairing via secret codes, cross-network messaging, poke/nudge, online status, read receipts, avatar exchange
- Distribution: dmg for macOS (arm64)
- Current version: v1.1.0

## Architecture
- **Client**: Tauri v2 app with Vite frontend (HTML/CSS/JS), Rust backend
  - Crypto: x25519-dalek (ECDH) + aes-gcm (AES-256-GCM)
  - Database: rusqlite (bundled SQLite) for local persistence
  - WebSocket: tokio-tungstenite for relay connection
  - Persistent UUID + private key stored in SQLite for reconnect protocol
- **Server**: Node.js WebSocket relay (ws library), handles pairing, relay, presence
  - identityRegistry Map for persistent identity across reconnects
  - reconnect, read_receipt, profile_update message handlers
- **Encryption flow**: Pairing exchanges public keys → ECDH shared secret → AES-GCM encrypt/decrypt
- **Pairing**: 6-digit codes, 5-minute expiry, generated by relay server
- **Reconnect**: Client sends persistentUUID on connect → server restores identity from registry
